How to use Syncitol
✕
How it works

Syncitol resolves each clip's record-start time from the best available source — the media's embedded creation time / timecode, read directly by the bundled FFmpeg decoder. If a file carries none, it falls back to the file modification date minus the clip's duration. Every clip is then placed on a new timeline so the gaps between recordings match real clock time, and the audio is fine-aligned by waveform.

No external install required — the FFmpeg audio decoder ships inside the plugin. The Detected Clips table tags each row meta (embedded metadata) or mtime (file date). On the mtime fallback, preserve file dates when copying media — if the OS updates mtime on copy the sync will be off.

Audio envelopes are cached on disk, so re-running a sync on unchanged media is much faster. When Premiere's own waveform cache (.pek peak files) covers the media, the coarse alignment reads it directly — milliseconds instead of minutes, no audio decoding.

⚡ Auto Sync

One click runs the whole pipeline hands-off: Scan → Build Sync → align (coarse whole-track + fine per-clip). Make your original sequence active first (not an already-built -SYNC — Auto Sync refuses those), then click Auto Sync. It stops with a clear message if a step can't proceed, and a ✕ Cancel button next to the spinner stops long audio scans at any time. When it finishes, the Sync Results table shows each track and clip's shift, the signal it matched on, and a confidence score. The Manual steps below run the stages individually — note the manual Fine Tune does the fast fine pass only, so Auto Sync is what corrects large clock offsets.

↺ Scan Sequence

Reads every clip's path, duration and type from your active sequence. Make the sequence you want to sync the active one first (double-click it in the Project panel).

⏱ Build Sync

Creates a new sequence named [original]-SYNC and places every clip at its real-clock-time position, preserving track layout and A/V links.

Files should be stored locally — network or cloud-synced paths may not report a reliable modification date.

≈ Fine Tune

The manual Fine Tune button runs the fine per-clip pass only (±5s) — fast, for when you're hands-on and have already nudged tracks into rough alignment. It auto-picks a reference track (the one with the most recorded coverage, on any track) and polishes every other clip to it via audio waveform cross-correlation. Only corrections larger than 20 ms are applied.

Large (minute-scale) clock offsets are handled by the coarse whole-track pass, which runs as part of ⚡ Auto Sync. If a manual Fine Tune leaves a track far off, run Auto Sync instead — or drag the track within ~5s first.

After a fine tune, ↩ Revert (in Sync Results) undoes exactly the shifts that were applied. On overlaps longer than 10 minutes the panel also checks for clock drift — devices whose clocks run at different rates — and warns with the measured drift (a single offset can't fix both ends; split long clips before syncing if flagged).
