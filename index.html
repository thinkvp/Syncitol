How to use DateModSync
How it works

DateModSync reads each clip's file modification date to determine when recording finished. Subtracting the clip's duration gives the approximate record-start time. Every clip is then placed on a new timeline so the gaps between recordings match real clock time.

Tip: preserve file dates when copying media — if the OS updates mtime on copy the sync will be off.

↺ Scan Sequence

Opens your active sequence and reads every clip's path, duration and type. Make the sequence you want to sync the active one first (double-click it in the Project panel).

⏱ Build Sync Sequence

Creates a new sequence named [original]-SYNC and places every clip at its real-clock-time position. Video and linked audio are placed together. Audio-only clips are routed to dedicated audio tracks.

Files must be stored locally — network or cloud-synced paths may not report a reliable mtime.

≈ Fine Tune Audio

Refines the rough mtime sync using audio waveform cross-correlation via ffmpeg. Just click the button — no configuration needed.

Each non-base track is compared against the tracks below it to find the best sub-second shift. Only corrections larger than 20 ms are applied.

How it works
±5sMaximum lag searched from current position
20sMax audio compared per pair (needs ≥3s overlap)
3×Samples overlap at 50%, 20% and 80% through the window
FastSkips extra windows when centred score ≥0.70

Requires ffmpeg on your system PATH. Run ffmpeg -version in a terminal to confirm it is available before using this feature.
